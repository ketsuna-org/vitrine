Confidentialite Modele de donnees local-first

Bot Creator garde les donnees bot en local par defaut et n'utilise le cloud que lorsque vous l'activez.

Le produit stocke par defaut les donnees bots, workflows, etat runtime et logs en local. Google Drive sert a la sauvegarde et a la restauration, pas a un backend heberge obligatoire.

Stockage local

Les donnees de votre bot restent sur l'appareil ou la machine qui execute le produit.

Bot Creator stocke metadonnees, reglages des commandes, workflows, variables, logs et donnees runtime dans le dossier local de l'application ou dans le volume monte du runner.

Sauvegardes optionnelles

Google Drive sert uniquement a la sauvegarde et a la restauration.

Quand vous connectez Google Drive, Bot Creator synchronise les sauvegardes dans AppData. Sur le runner, les tokens OAuth restent dans le volume monte pour survivre aux redemarrages.

Diagnostics et logs

Les echecs runtime ecrivent des diagnostics en local.

Les diagnostics de demarrage sont ecrits localement pour eviter les echecs silencieux. Sur le runner, les logs sont ecrits dans le fichier persistant du volume.

Votre responsabilite

Protegez exports, sauvegardes et comptes relies.

Les bot tokens sont des secrets sensibles. Securisez l'appareil ou le serveur qui execute le produit, protegez vos sauvegardes et le compte Google utilise pour l'acces backup.

Contact
