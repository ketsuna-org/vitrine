Confidentialite Modele de donnees local-first

Bot Creator garde le workflow principal en local et n'utilise le cloud que lorsque vous le demandez.

Le produit stocke par defaut les donnees bots, workflows, etat runtime et logs en local. Google Drive sert aux flux de sauvegarde et de restauration, pas a un backend heberge obligatoire.

Stockage local

Les donnees de votre bot restent sur l'appareil ou la machine qui execute le produit.

Bot Creator stocke metadonnees, configurations de commandes, workflows, variables, logs et donnees runtime dans le dossier local de l'application ou dans le volume monte du runner.

Sauvegardes Google Drive

Drive sert uniquement a la sauvegarde et a la restauration.

Quand vous connectez Google Drive, Bot Creator synchronise les sauvegardes dans AppData. Sur le runner Docker, les tokens OAuth persistent dans le volume monte pour survivre aux redemarrages.

Diagnostics et logs

Les echecs runtime ecrivent des diagnostics en local.

Les diagnostics de demarrage sont ecrits localement pour eviter les echecs silencieux. Sur le runner, les logs sont ecrits dans le fichier persistant du volume.

Responsabilite partagee

Vous restez responsable de la securite des exports et des comptes relies.

Les bot tokens sont des secrets sensibles. Protegez vos sauvegardes, l'appareil ou le serveur qui execute le produit, et le compte Google utilise pour les sauvegardes.

Ce que Bot Creator ne fait pas

  • Le produit ne requiert pas de backend heberge dedie pour creer et executer des bots.
  • Le produit n'impose pas de synchronisation cloud dans le workflow par defaut.
  • La configuration OAuth Google Drive n'est pas incluse par defaut dans l'image Docker prebuild.
