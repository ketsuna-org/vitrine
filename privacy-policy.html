Privacy Local-first data model

Bot Creator keeps the primary workflow local and uses cloud backup only when you ask for it.

The product stores bot data, workflows, runtime state, and logs locally by default. Google Drive is used for backup and restore flows, not as a mandatory hosted backend.

Local storage

Your bot data stays on the device or machine running the product.

Bot Creator stores metadata, command configurations, workflows, variables, logs, and runtime-related data inside the local app documents directory or the mounted runner volume.

Google Drive backups

Drive is used for backup and restore only.

When you connect Google Drive, Bot Creator syncs backup data to the AppData folder. On the Docker runner, OAuth tokens persist inside the mounted volume so they survive container restarts.

Diagnostics and logs

Runtime failures write diagnostics locally.

Startup diagnostics are written locally so failed initialization does not disappear silently. On the runner, logs are written to the volume-backed log file.

Shared responsibility

You remain responsible for protecting exports and connected accounts.

Bot tokens are sensitive secrets. Protect backups, secure the device or server running the product, and secure the Google account used for backup access.

What Bot Creator does not do

  • It does not require a dedicated hosted backend to create and run bots.
  • It does not force cloud sync for the default workflow.
  • It does not include Google Drive OAuth configuration in the prebuilt Docker image by default.
