Privacy Local-first data model

Bot Creator keeps bot data local by default and uses cloud backup only when you enable it.

The product stores bot data, workflows, runtime state, and logs locally by default. Google Drive is available for backup and restore, not as a required hosted backend.

Local storage

Your bot data stays on the device or machine running the product.

Bot Creator stores metadata, command settings, workflows, variables, logs, and runtime data in the local app directory or the mounted runner volume.

Optional backups

Google Drive is used for backup and restore only.

When you connect Google Drive, Bot Creator syncs backup data to the AppData folder. On the runner, OAuth tokens stay in the mounted volume so they survive restarts.

Diagnostics and logs

Runtime failures write diagnostics locally.

Startup diagnostics are written locally so failed initialization does not disappear silently. On the runner, logs are written to the persistent volume-backed log file.

Your responsibility

Protect exports, backups, and connected accounts.

Bot tokens are sensitive secrets. Secure the device or server running the product, protect your backups, and secure the Google account used for backup access.
