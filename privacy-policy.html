Privacy Local-first data model

Bot Creator keeps the primary workflow local and uses cloud backup only when you ask for it.

The product stores bot data, workflows, logs, and runtime state locally by default. Google Drive is used for backup and restore flows, not as a mandatory hosted backend.

Local storage

Bot data lives on the device or machine where you run the product.

Bot Creator stores app metadata, command configurations, workflows, variables, logs, and runtime-related data under the local app documents directory or the mounted runner volume.

Google Drive backups

Google Drive is used for backup and restore flows only.

When you connect Drive, Bot Creator syncs backup data to the AppData folder. On the Docker runner, OAuth tokens are persisted inside the mounted volume so they survive container restarts.

Diagnostics and logs

Runtime failures write diagnostics locally.

Startup diagnostics are written locally so failed initialization does not disappear silently. On the runner, logs are written to the volume-backed log file.

Shared responsibility

You remain responsible for securing exported data and connected accounts.

Bot tokens are sensitive secrets. Treat backups and exports carefully, protect the device or server where the product runs, and secure the Google account used for backup access.

What Bot Creator does not do

  • It does not require a dedicated hosted backend to create and run bots.
  • It does not force cloud sync for the default workflow.
  • It does not include Google Drive OAuth configuration in the prebuilt Docker image by default.
